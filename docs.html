Overview

mimo is a yield optimization protocol that routes user deposits across multiple chains to capture the highest available yield. It uses an intent-based architecture where solvers compete to find optimal routes.

Key concept: Users never interact with bridges directly. The solver network handles cross-chain settlement atomically.

Quickstart

Get started with mimo in 3 steps:

1

Install the SDK

npm install @mimo-protocol/sdk
2

Initialize the client

import { MimoClient } from '@mimo-protocol/sdk';

const mimo = new MimoClient({
  chainId: 1, // Ethereum
  rpcUrl: process.env.RPC_URL,
});
3

Find best yield & deposit

const bestYield = await mimo.findBestYield({
  token: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48', // USDC
  amount: ethers.parseUnits('1000', 6),
});

const tx = await mimo.deposit(bestYield.vault, amount);
await tx.wait();

Architecture

mimo consists of three core layers:

Layer 1

Vault Contracts

User-facing contracts on each chain. Handle deposits, withdrawals, and share accounting. These are the only contracts users interact with.

Layer 2

Solver Network

Off-chain operators who compete to find optimal yield routes. They submit bids for user intents and execute the best route atomically.

Layer 3

Settlement Layer

Cross-chain messaging using LayerZero + custom ZK proofs. Ensures atomic settlement without relying on traditional bridges.
